System Overview

What the Academic Administration System does

The Academic Administration System (AAS) is a web-based platform for managing academic records including certificates, subjects, and student groups. It provides role-based access for regular users and administrators.

📚

Subjects by Semester

Browse subjects available in each semester by selecting from a dropdown. Three semesters available with multiple subjects each.

📜

Certificates

Add, edit, and delete academic certificates. Each certificate has a type (Study Certificate or Academic Leave) and a company/institution name.

👥

Group Students

View students organized by group. Select a group to see its members, and remove students if needed.

🔐

Admin Panel

Administrator-only access to manage all users: view accounts and change user roles between User and Administrator.

User Authentication

Sign Up, Sign In, and Sign Out

Creating a New Account (Sign Up)

Navigate to http://localhost:3000/signup or click the Sign Up link in the top navigation.

1
Username — Enter your display name (minimum 2 characters)
2
Email address — Enter a valid email (e.g. john@example.com)
3
Password — Enter a password between 8 and 32 characters
4
Click "Add" — Your account is created and you are redirected to the home page
⚠️ New accounts default to "user" role To get administrator access, an existing administrator must change your role in the Admin panel, or you can update the role directly in MongoDB.

Signing In

Navigate to http://localhost:3000/signin or click the Sign In link.

1
Email address — Enter your registered email
2
Password — Enter your password
3
Click "Add" — You are logged in and redirected to the home page
✅ Session Duration Your session lasts 1 hour. After that you will need to sign in again. The session is stored in a secure browser cookie.

Signing Out

1
Look at the top-right of any page when you are logged in
2
Click the Sign Out button
3
You are automatically redirected to the Sign In page

Validation Rules

FieldRule
UsernameMinimum 2 characters
EmailMust be a valid email format
Password8–32 characters
🔒 Security Note Passwords are encrypted using bcrypt before being stored in MongoDB. Your plain-text password is never saved.

Subjects by Semester

Browse subjects available in each academic semester

The Subjects page lets you view which subjects are taught in each semester. Simply select a semester from the dropdown and the subjects will load automatically.

URL: http://localhost:3000/subjects

How to View Subjects

1
Go to the Subjects page — click Subjects in the navigation bar or visit the URL above
2
Select a semester from the dropdown menu (First, Second, or Third)
3
The subject list updates automatically — no need to click a button
ℹ️ Note If no semester is selected, the subject list will be empty. Subjects only appear after you pick a semester.

Available Subjects by Semester

SemesterSubjects
First Math, English
Second Object-Oriented Programming, Professional Language
Third Networks, Databases

Behind the Scenes (How It Works)

When you select a semester, the page sends a request to the API:

  • Semesters are loaded from /api/semesters when the page opens
  • Selecting a semester calls /api/subjects?semesterId=1 (or 2 or 3)
  • The subject list re-renders with the returned subjects

Certificates Management

Add, view, edit, and delete academic certificates

The Certificates page is a full CRUD (Create, Read, Update, Delete) feature. You can manage academic certificates that are stored in MongoDB.

URL: http://localhost:3000/certificates

Adding a New Certificate

1
Click the "+" (Plus) button — a modal form opens on the right side of the screen
2
Select Certificate Type from the dropdown:
Study Certificate or Academic Leave
3
Enter Company/Institution Name (minimum 2 characters)
4
Click "Add" — the certificate is saved and appears in the list immediately

Viewing Certificates

All saved certificates appear in a table on the Certificates page with these columns:

ColumnDescription
NameThe certificate type (Study Certificate or Academic Leave)
NoteThe company or institution name
ActionsEdit (pencil ✏️) and Delete (trash 🗑️) buttons

Editing a Certificate

1
Find the certificate in the table and click the pencil ✏️ icon
2
The form opens with the current values pre-filled
3
Change the type or company name as needed
4
Click "Add" — the record is updated in MongoDB

Deleting a Certificate

1
Find the certificate in the table
2
Click the trash 🗑️ icon
3
The certificate is deleted immediately — the list refreshes automatically
⚠️ Deletion is permanent There is no confirmation dialog. Once you click the trash icon, the certificate is permanently deleted from MongoDB.

Admin Panel — User Management

Administrator-only access to manage system users
🔒 Restricted Access This page is only accessible to users with the administrator role. Regular users are automatically redirected away. The Admin link only appears in navigation for administrators.

URL: http://localhost:3000/admin/users

What Administrators Can Do

  • View a table of all registered users in the system
  • See each user's name, email address, and current role
  • Change any user's role between user and administrator

Changing a User's Role

1
Sign in with an administrator account
2
Click "Admin" in the top-right navigation bar
3
Find the user in the table
4
Use the role dropdown next to the user's name to select their new role
5
The role is saved immediately — no submit button needed

User Table Columns

ColumnDescription
NameUser's display name (set during registration)
EmailUser's email address
RoleDropdown to view and change role

Making the First Administrator (Manual MongoDB Method)

If no administrator exists yet, follow these steps to promote a user manually:

1
Open MongoDB Atlas → your cluster → Browse Collections
2
Find the users collection in the academic-certification database
3
Find your user document and click Edit
4
Add the field: "role": "administrator"
5
Click Update — sign out and sign back in to apply
✅ Quick Method — Use Seed Users Visit http://localhost:3000/api/seed-user to automatically create admin@test.com with administrator role.

Troubleshooting

Common problems and how to fix them

MongoDB Connection Errors

Error: querySrv ECONNREFUSED Your network's DNS server is blocking MongoDB SRV lookups.

Fix: Change your DNS server to Google DNS: Go to Settings → Network & Internet → your connection → DNS → set to 8.8.8.8 Then restart npm run dev.
Error: MongoServerError: bad auth / Authentication failed The username or password in your .env.local is wrong.

Fix: Go to MongoDB Atlas → Database Access → check your user credentials match the MONGODB_URI in .env.local.
Error: connection timed out / ETIMEDOUT Your IP address is not whitelisted in MongoDB Atlas.

Fix: Go to MongoDB Atlas → Network Access → Add IP Address → Add Current IP Address.

Application Issues

ProblemCauseFix
No subjects appear after selecting semester API not responding Check server is running with npm run dev
No groups in Group Students dropdown Database not seeded Visit /api/seed to populate data

Changes not appearing after edit Browser cache Hard refresh: Ctrl + Shift + R

Working on Different Networks (Home vs Office)

If the app works at one location but not another:

  • IP Whitelist — Each network has a different public IP. Add your current IP to MongoDB Atlas Network Access
  • DNS Blocking — Some ISPs block SRV DNS lookups. Change DNS to 8.8.8.8 (Google) or 1.1.1.1 (Cloudflare)
  • Port 27017 blocked — Some routers block this port. Test with: Test-NetConnection -ComputerName <host> -Port 27017

Quick Checklist for First-Time Setup

  • .env.local file exists with all 5 required variables
  • Your IP is added to MongoDB Atlas Network Access
  • DNS is set to 8.8.8.8 (if on a network that blocks SRV lookups)
  • Dev server is running: npm run dev
  • Database seeded: visited /api/seed and /api/seed-user once
